# STM32 语音交互助手项目说明

本项目基于野火 STM32F103VET6 开发板搭建，实现脱离上位机的独立语音对话设备。终端主要负责屏幕 UI 绘制、音频数据采集与声音外放。复杂的语音识别、大模型对话请求、语音合成以及历史记录管理等核心任务，均通过网络透传交由云端代理服务器处理。

## 系统架构与组成清单

* 主控终端：野火 STM32F103VET6 开发板，系统已配置外部 SPI Flash 中文字库与 FSMC 接口 LCD 彩色屏幕。
* 网络通信：ESP8266 串口 Wi-Fi 模块，推荐使用 ESP-01S 规格。
* 语音输入：INMP441 全向数字麦克风模块，采用 I2S 数字接口。
* 语音输出：MAX98357A 数字音频功放模块，搭配 3W 小型扬声器。
* 云端管理：基于 Flask 配合 Vue3 构建的 WebUI 服务端，负责对话历史持久化存储、API 节点配置以及终端设备管理。

## 终端状态机与按键交互逻辑

系统交互全部操作由开发板自带的 K1 和 K2 两个物理按键完成。底层定时器负责按键消抖并区分长按、短按与双击事件。系统核心包含主对话界面与历史记录列表两大运行状态。

### 主对话界面交互逻辑

**K1 键：语音录入与分段拼接**
* 按住不放：系统进入录音采集状态。单片机开启 I2S 和 DMA 外设流式采集麦克风音频，屏幕同步显示麦克风录音动效。录音数据实时通过 Wi-Fi 透传至云端缓存。
* 松开按键：录音暂停，单片机停止采集音频。用户可以再次按住 K1 键继续说话，实现多次间断输入的音频分段拼接。

**K2 键：对话控制与状态切换**
* 录音暂停时短按：确认发送。向云端发送结束标识帧，通知服务器音频输入完毕，云端正式开始拼接音频并调用大模型生成返回结果。
* 录音暂停时长按：放弃发送。系统立刻清空本地与云端当前待处理的录音缓存数据，强制取消本次输入并返回待机状态。
* 待机状态短按：开启新会话。系统向云端服务器发送创建新会话指令，云端将前次对话归档存入数据库，同时为终端分配全新的上下文环境，屏幕同步显示新对话已开启提示。
* 播放状态短按：打断当前播报。设备播放语音时按下此键可立即停止音频输出并清空接收缓存，使设备快速回到待机状态准备下一次输入。
* 待机状态双击：打开历史对话界面。系统向云端请求历史对话摘要列表并渲染至屏幕，设备运行状态随之切换至历史记录列表模式。

### 历史记录列表模式交互逻辑

在此模式下，系统暂停语音收发功能，按键功能转为纯粹的菜单导航控制。

* K1 键短按：列表光标向上移动一项。
* K2 键短按：列表光标向下移动一项。
* 任意键双击：确认载入选中记录。系统向云端发送加载该对话上下文指令，屏幕刷新加载对应聊天内容，随后设备自动退出列表模式恢复至主对话界面。

## 数据流转与云端管理机制

上行数据方向：单片机通过 I2S 接口获取 16kHz 采样率与 16bit 位深的原始音频流。借助 DMA 半满和全满中断将切片数据搬运至高速串口，ESP8266 模块随后通过 TCP 协议将纯音频裸流推送到云端代理服务器。断续录音期间云端暂存所有接收到的音频切片，直至收到发送指令后再进行拼接识别。

下行数据方向：云端代理服务器依次完成语音转文字、请求大模型、文字转语音的全部流程，并将完整对话存入数据库。服务器随后将转码后的中文文本与合成后的音频数据流下发。单片机接收后直接提取文本用于屏幕 UI 刷新，并将音频流数据送入功放模块进行外放。

WebUI 管理后台：用户可通过浏览器访问基于 Flask 与 Vue3 搭建的管理后台，可视化查看所有终端上传的对话记录，管理历史会话，配置大模型 API 密钥以及调整系统运行参数。

## 开发要求

* 代码规范：所有代码和文件必须遵循统一的命名规范和注释风格，确保可读性和维护性。
* 模块化设计：各功能模块应放在不同文件夹中，分成不同文件，封装好功能接口。**单个文件不宜过长**。
